AI as Colleague

도구가 아닌 동료가 된 순간

나: "이 규칙 목록 보고 검증 코드 짜줘"

Claude: "택셀네트컴/텍셀네트컴 표기 불일치는 KOS 전산에 이미 그렇게 등록된 거니까 수정하면 안 되겠죠. 그대로 두겠습니다."
나: "FastAPI 백엔드로 만들어줘"

Claude: "규칙 15개 다 명확한 조건 분기예요. 굳이 서버 안 써도 됩니다. 브라우저만으로 구현하면 보안 이슈도 없어요."
1

CLAUDE.md에 규칙을 썼더니

내가 설명 안 해도 맥락을 파악하고 코드를 짰다

2

피벗을 제안해왔다

FastAPI → 순수 HTML. 서버 비용 0, 보안 이슈 0

3

예외를 스스로 발견했다

택셀/텍셀 오타처럼 보이지만 전산 등록값 — 건드리면 안 됨

4

선을 지켰다

"사업 특성 판단은 담당자가 해야 합니다" — 스스로 한계를 선언

Role Expansion

전에는 못했던 것을
할 수 있게 됐다

Before Agent
❌ 개발자 없이 웹 앱 불가능
❌ 규칙 바뀌면 문서만 수정
❌ 협력사 셀프 검증 불가
❌ 기획자 = 검증 전담 인력
❌ 오류 유형 파악 불가
→
After Agent
✅ 단일 HTML 파일로 웹 서비스 완성
✅ CLAUDE.md 수정 → 코드 자동 반영
✅ 협력사가 스스로 업로드·검증·수정
✅ 담당자는 최종 승인만 담당
✅ 오류 패턴 통계 확장 가능

기획자가 개발자 없이 3주 만에 프로덕션 수준 웹 서비스 완성

Tacit Knowledge

머릿속에만 있던 것을
CLAUDE.md에 담았다

1
벤더사 ↔ 공급사명 1:1 매핑18쌍. 공백 하나도 틀리면 오류
2
기본연도 9개 허용값무약정·반납·완불 조합만 허용
3
기본연도 → 리스사명 자동 연계완불=KT커머스, 반납=AJ네트웍스
4
상품 유형별 가격 칸 규칙채울 칸/비울 칸이 유형마다 다름
5
제품명 형식 3가지[보안] / [보안][SA] / [보안-CarePack]
9
서비스번호 입력 여부 — 사람 판단AI는 존재 확인만, 사업 특성은 담당자가
11
CarePack 단일 통합 등록제품이 여러 개여도 1행으로 묶어야
15
양식 구조 훼손 감지표준 63열 대비 누락 시 재작성 요청
